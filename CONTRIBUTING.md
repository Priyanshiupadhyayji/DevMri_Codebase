# Contributing to DevMRI

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/devmri-app.git`
3. Create a branch: `git checkout -b feature/your-feature-name`

## Development Setup

```bash
npm install
npm run dev
```

## Code Style

- Use TypeScript for all new code
- Follow the existing code conventions
- Run `npm run lint` before committing

## Commit Messages

This project uses [Conventional Commits](https://www.conventionalcommits.org/). Please use the following format:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `refactor:` for code refactoring
- `test:` for adding tests
- `chore:` for maintenance tasks

## Pull Request Process

1. Ensure all tests pass and linting is clean
2. Update documentation for any changed functionality
3. Request review from maintainers
4. PRs require at least one approval before merging
